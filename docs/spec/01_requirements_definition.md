# 要件定義書: D.scout

## 1. システム概要
同志社大生・同女大生が自分にぴったりのコミュニティ（サークル・ゼミ）から「スカウト」を受けることで、新しい繋がりを発見できるプラットフォーム。

## 2. 機能要件

### 2.1 認証・セキュリティ
*   **大学ドメイン制限**:
    *   `@doshisha.ac.jp` または `@mail.doshisha.ac.jp` のみを許可。
    *   正規表現によるフロントエンド/バックエンドでの二重チェック。
*   **パスワードバリデーション**: 6文字以上必須。

### 2.2 ホーム (団体発見)
*   **検索**: インクリメンタルなフリーワード検索。
*   **フィルタリング**: ジャンル（スポーツ、文化系、学術など）によるチップ選択。
*   **UI表示**:
    *   2列グリッドのカード形式。
    *   キャンパス名（今出川/京田辺）をバッジで表示。
    *   団体名、ロゴ、ジャンルタグ、簡潔な紹介文。

### 2.3 スカウト (マッチング)
*   **通知**: 未読スカウトに対するレッドドット（赤丸）表示。
*   **承認フロー**:
    *   「承認」ボタンタップで既読化。
    *   団体公式Instagramへのディープリンク遷移（別タブ）。
*   **履歴**: 時系列（ListView）表示、相対時間表示（今日/昨日など）。

### 2.4 イベント (スケジュール)
*   **表示形式**: 開催日ごとのセクション分割。
*   **キャンパスカラー識別**:
    *   今出川: 紫 (#800080)
    *   京田辺: 緑 (#008000)
    *   両キャンパス: 青 (#0000FF)
*   **カレンダー連携**: 標準カレンダーアプリへの登録導線（モック）。

### 2.5 マイページ (学生プロフィール)
*   **属性表示**: アバター、名前、学部、学年、メインキャンパス。
*   **タグ管理**:
    *   「興味・関心」をチップ形式で管理。
    *   編集モードでの削除・新規追加機能。
*   **メニュー**: ログアウト、通知設定、プロフ編集。

## 3. 非機能要件
*   **UI/UX**: 
    *   ONE CAREERスタイル（白基調、清潔感、洗練）。
    *   アクセントカラー: Indigo-600。
    *   アニメーション: `motion/react` (またはFlutter同等のアニメーション) を使用した滑らかな遷移。
*   **レイアウト**: 
    *   最大幅 `max-w-md` (448px) 内にコンテンツを収める。
    *   ボトムバー固定（4タブ）。
*   **パフォーマンス**: インクリメンタル検索の低レイテンシ。

## 4. デザインシステム (Design Tokens)
*   **Background**: `bg-gray-50`, `bg-white`
*   **Primary**: `indigo-600`
*   **Text**: `slate-900` (メイン), `slate-500` (サブ)
*   **Border**: `slate-200`
