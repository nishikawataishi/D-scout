# テスト計画書: D.scout

## 1. テストの基本方針
プログラム未経験の学生でも実行・確認できる「挙動確認（手動テスト）」をメインとし、セキュリティ上重要な「認証ロジック」については自動単体テスト（Unit Test）を導入します。

## 2. 単体テスト (Unit Test - 自動)
*   **認証バリデーション**:
    *   `@doshisha.ac.jp` → OK
    *   `@mail.doshisha.ac.jp` → OK
    *   `@gmail.com` → NG
    *   `@ritsumei.ac.jp` → NG
    *   パスワード 5文字 → NG
    *   パスワード 6文字 → OK

## 3. 結合テスト (Integration Test - 手動確認)
モックアップ画面での操作性を中心に、以下のフローを確認します。

| カテゴリ | テスト項目 | 期待される結果 |
| :--- | :--- | :--- |
| **認証** | 不適切なドメインでのログイン試行 | エラーメッセージが表示され、ホームに遷移しない。 |
| **ホーム** | ジャンルチップのタップ | 指定したジャンルの団体のみがリストに表示される。 |
| **スカウト** | 承認ボタンのタップ | レッドドットが消え、Instagram（モック）が開く。 |
| **イベント** | カレンダーアイコンのタップ | 「カレンダーに追加しました（モック）」とアラートが出る。 |
| **マイパ** | タグの削除（×タップ） | タグがアニメーションを伴って消える。 |

## 4. ユーザー受け入れテスト (UAT)
西川様（オーナー）が実機またはエミュレーターで操作し、「ONE CAREER風の清潔感」と「直感的な操作感」が実現されているかを確認します。

## 5. 合否判定基準
1. 同志社ドメイン以外のログインが完全にブロックされていること。
2. 4つの主要タブが正常に切り替わり、定義された要素が表示されていること。
3. エラーが発生した際に、不適切なメッセージ（プログラムの生エラー）が出ず、日本語で説明が表示されること。
